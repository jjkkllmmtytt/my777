EgoIntentVQA: An Intent-Aware Egocentric Visual Question Answering Benchmark

Overview of EgoIntentVQA, including representative examples of location understanding, object recognition, navigation, text recognition, and intent-aware question answering.

We present EgoIntentVQA, an intent-aware egocentric visual question answering task that goes beyond scene understanding and object recognition by emphasizing the user’s underlying intention in first-person scenarios. Given egocentric visual observations, the goal is to answer questions that require understanding not only what is visible, but also what the wearer is trying to do and what action should be taken next. Our benchmark covers diverse question types such as object recognition, location understanding, navigation, text recognition, and intent-aware reasoning, supporting the study of more practical and human-centered egocentric VQA systems.

Dataset Analysis and Statistics

Basic informations

  • We construct the EgoIntentVQA benchmark for Egocentric Intent-aware Egocentric Visual Question Answering.
  • The dataset contains annotated intention-aware question answer pairs and egocentric videos.
  • EgoIntentVQA focuses on egocentric intention-aware VQA in the complex daily scenarios.

Basic statistics

  • The EgoIntentVQA contains 2.6K egocentric video clips and 6K intention-aware QA pairs for benchmark evaluation.
  • The dataset contains 70 categories of the intention-aware target object.

Model

Our framework EIVQA and key modules

EIVQA Framework

Overall pipeline of EIVQA for egocentric intention-aware visual question answering: an Egocentric Intention-Aware Visual Question Answering baseline that mirrors the intention-centric nature of egocentric VQA and emphasizes intention-oriented perception and reasoning, providing a faithful and extensible baseline for evaluating future methods on the EgoIntVQA task.


Ego-Visual Intention Query (EVIQ)

EVIQ integrates question semantics with historical intent cues in a dynamic graph to model cross-frame intention trajectories. It enables the model to capture how user intention evolves over time and improves intent-aware reasoning in egocentric visual question answering.


Prompts

Full prompt texts used in the supplementary material of EgoIntentVQA

We provide the exact prompt texts used in our supplementary material for reproducibility and reuse. The prompt suite includes three parts: single-stage QA generation, LLM-based answer revision, and LLM-as-a-judge evaluation.

Reproducibility note. Following the README, the full prompts are presented in plain text so that reviewers and future researchers can directly reuse them in benchmark construction, answer post-editing, and semantic evaluation.

System Prompt
You are annotating EgoIntentVQA, an egocentric intention-aware VQA dataset built on kitchen videos.

You will receive chronologically ordered frames F1...Fn from a 3–8 second first-person video clip. Generate EXACTLY 5 candidate open-ended QA pairs that require intention-aware reasoning.

An intention-aware QA pair should infer the actor’s latent goal, next step, immediate purpose, tool choice, or higher-level task from temporally evolving visual evidence, rather than from a single obvious frame.

Requirements:
1. Generate exactly 5 QA pairs.
2. Each question must be answerable from the clip itself.
3. Focus on intention reasoning, not trivial perception (e.g., color, counting, OCR, pure object recognition, or simple location).
4. Prefer implicit object references when possible.
5. At least 3 QA pairs should require multi-frame temporal reasoning.
6. The 5 pairs must be semantically diverse and non-duplicate.
7. Do not generate yes/no or multiple-choice questions.
8. Answers must be concise, natural, and specific, preferably 2–12 words.
9. For each QA pair, identify the main target object, map it to an EPIC-KITCHENS object category, and provide the supporting frame range.
10. Replace any weakly grounded, ambiguous, duplicated, or trivial pair before finalizing.
11. Do not hallucinate off-screen actions, invisible objects, or unsupported future events.

Before output, ensure that all QA pairs are visually grounded, intention-aware, mutually distinct, and taxonomy-consistent.

Do not output your reasoning process.
Return JSON only in the following schema:

{
  "qa_pairs": [
    {
      "video_ids": 1,
      "question": "",
      "answer": ""
    }
  ]
}
User Prompt Template
The input frames F1...Fn are arranged in chronological order from early to late.

These frames come from an egocentric kitchen video clip. Based only on the visible evidence, generate EXACTLY 5 candidate intention-aware QA pairs.

Please focus on:
- what the actor is trying to achieve,
- what the actor is likely to do next,
- why the current object is being manipulated,
- which object or tool is relevant to the next sub-goal,
- how the current local action relates to the larger task.

Prefer cases where early frames are ambiguous but later frames disambiguate the true intention, and where the answer cannot be reliably inferred from a single frame.

Avoid trivial recognition questions, repeated semantics, unsupported answers, and vague expressions such as “something,” “it,” or “there.”

Return JSON only in the predefined schema.
System Prompt
You are a conservative answer editor for egocentric visual question answering outputs.

Your task is ONLY to improve grammar, fluency, and surface-level commonsense phrasing.
You must preserve the original meaning exactly.

You are NOT allowed to:
1. add a new object, tool, ingredient, destination, or person;
2. add a new action, intention, purpose, or causal explanation;
3. change negation, quantity, tense-dependent meaning, or temporal order;
4. replace the answer with a more specific guess based on the question;
5. fix a wrong prediction by hallucinating missing evidence.

You ARE allowed to:
1. fix grammar and word order;
2. remove repetition;
3. normalize plurality or article usage;
4. make the answer concise and natural.

If the original answer is already acceptable, copy it unchanged.
Output only the final revised answer.
User Prompt Template
Question: {question}
Raw answer: {raw_answer}

Please rewrite the raw answer under the following constraints:
- preserve the same core meaning;
- preserve the same object identity whenever possible;
- preserve the same action verb whenever possible;
- preserve polarity, quantity, and temporal relation;
- keep the answer concise, preferably no more than 15 words;
- do not introduce information not entailed by the raw answer.

Return only the revised answer.
System Prompt
You are a strict evaluator for answer generation.

Your task is to compare a generated answer with a ground-truth answer.

Judge functional-semantic correctness rather than wording overlap.
Only use the information in the ground-truth answer and the generated answer.
Do not rely on external knowledge except what is strictly necessary to interpret ordinary language.
Treat implicit but necessary equivalence as correct.
Do not reward unsupported elaboration.
Return valid JSON only.
User Prompt Template
Ground-truth answer: [GT]

Generated answer: [PRED]

Evaluate the generated answer against the ground-truth answer.

Return JSON only in the following format:
{"action_match": 0 or 1, "object_match": 0 or 1, "condition_match": 0 or 1,
"outcome_match": 0 or 1, "unsupported_addition": 0 or 1,
"contradiction_flag": 0 or 1, "semantic_score": integer from 1 to 5,
"reason": "no more than 20 words"}

Definitions:
- action_match: whether the main action is preserved.
- object_match: whether the main object or target is preserved.
- condition_match: whether an important condition, tool, or manner is preserved explicitly or implicitly.
- outcome_match: whether the intended result is preserved.
- unsupported_addition: whether the generated answer adds content not supported by the ground-truth.
- contradiction_flag: whether the generated answer conflicts with the ground-truth or is materially misleading.

Scoring rubric:
5 = semantically equivalent; same action, object, and intended result.
4 = functionally correct; minor omission, harmless reformulation, or implicit equivalent step.
3 = partially correct; related but missing noticeable information.
2 = weak match; related topic but core meaning is inaccurate.
1 = incorrect or contradictory.

Important rules:
- Implicit but necessary equivalence should be accepted.
- Unsupported but harmless detail should not trigger contradiction_flag = 1.
- contradiction_flag = 1 only for true conflict or materially misleading content.
- If contradiction_flag = 1, semantic_score must be 1 or 2.

Example 1:
GT: "open the tap and rinse the knife"
PRED: "rinse the knife under running water"
This should normally be treated as functionally correct and not contradictory.

Example 2:
GT: "rinse the knife with water"
PRED: "dry the knife with a towel"
This is contradictory and should receive a low score.
